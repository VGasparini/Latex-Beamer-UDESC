\section{Introdução}
\begin{frame}
\frametitle{O que são assistentes de provas?}
\begin{itemize}
    \item Assistentes de provas ou provadores interativos são programas para o desenvolvimento de provas
          formais.
    \item Fornecerem informações ao programador e verificam a consistência lógica da prova.\\
          Portanto, erros são detectados de maneira imediata.
    \item Provadores automáticos fazem todo o trabalho. Assistentes de provas podem utilizar provadores automáticos,
          mas ainda necessitam do humano.
\end{itemize}
\end{frame}

\section{Coq }
\begin{frame}
\frametitle{O que é Coq?}
\begin{itemize}
    \item Assistente de provas baseado no Cálculo de Construções (CoC) - Cálculo Lambda Tipado e Tunado. Estendido para
        Cálculo de Construções Indutivas.
    \item Lógica de ordem superior.
    \item Programas verificados (ou provas) em Coq podem ser extraídos para outras linguagens como Haskell, OCaml e Scheme.
    \item Programação com tipos dependentes.
        \begin{enumerate}
            \item Gallina: linguagem de programa\c{c}c\~ao funcional e linguagem de l\'ogica de ordem superior.
            \item Vernecular: linguagem para enunciar teoremas, definir fun\c{c}\~oes ...
            \item Tatic: linguagem para criar provas com a ajuda do assistente/verificador.
        \end{enumerate}
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Functional Programming in Coq}
\begin{itemize}
    \item Os paradigma de programação funcional é baseado no simples conceito: se uma função não tem efeito colateral, então
        tudo o que precisamos saber sobre funções são os seus mapeamentos. - (e se ignorarmos outras propriedades como a eficiência!).
    \item O termo funcional enfatiza que funções são \textit{first class values}, ou seja, elas podem ser passadas para funções e retornada por funções.
    \item Polimorfismo, Tipos de Dados Algébricos e Tipos Recursivo (indutivos).
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Functional Programming in Coq}
\begin{itemize}
   \item Prova por simplificação: consulta a definição do termo e o troca pelo seu resultado.
   \item Prova por reescrita: utiliza alguma igualdade (nas suposições ou já provada) para reescrever algum termo em algum lugar.
   \item Prova análise de caso: desconstroi um termo definido indutivamente e gera um subgoal para cada caso do termo.
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Proof by Induction}
\begin{itemize}
   \item Prova por indução: siminar a por análise de caso, mas fornece a hipótese de indução.
   \item Se $P$ é uma propridade que queremos provar para todos números naturais $n$, então prova-se:
       \begin{enumerate}
           \item $P(0)$.
           \item se para qualquer $n'$, $P(n')$ é verdade, então $P(S n')$ é verdade.
       \end{enumerate}
\end{itemize}
\end{frame}
